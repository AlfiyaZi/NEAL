Tutorial: Writing a new rule¶

Structure¶

NEAL’s Domain Specific Language (DSL) is designed to match the Abstract Syntax Tree (AST) of the program being analysed as closely as possible.

The rules have the following format:

rule RuleName {
  ProviderName::AstNode where Condition {
    AnotherAstNode where Condition {
      action()
    }
  }

  /* ... */
}

To make it more concrete, take a look at the couple examples that follows.
